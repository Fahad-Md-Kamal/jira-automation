Jira Push Hook Packaging And Release Instructions

Build package locally:

1) Install build tools:
.venv/bin/python -m pip install --upgrade build twine

2) Clean previous artifacts:
rm -rf dist build src/*.egg-info

3) Build source and wheel:
.venv/bin/python -m build

4) Validate metadata:
.venv/bin/python -m twine check dist/*

Publish to TestPyPI:

5) Configure ~/.pypirc with API tokens:
[distutils]
index-servers =
    testpypi
    pypi

[testpypi]
username = __token__
password = pypi-<your-testpypi-token>

[pypi]
username = __token__
password = pypi-<your-pypi-token>

6) Upload artifacts to TestPyPI:
.venv/bin/python -m twine upload -r testpypi dist/jira_push_hook-<version>-py3-none-any.whl dist/jira_push_hook-<version>.tar.gz

7) Verify install from TestPyPI:
.venv/bin/python -m pip install --index-url https://test.pypi.org/simple/ --no-deps jira-push-hook
jira-push-hook --help

Publish to PyPI:

8) Upload artifacts to production PyPI:
.venv/bin/python -m twine upload dist/jira_push_hook-<version>-py3-none-any.whl dist/jira_push_hook-<version>.tar.gz

9) Verify install from PyPI:
.venv/bin/python -m pip install --upgrade jira-push-hook
jira-push-hook --help

Release rules:
- Increase version in pyproject.toml before every new release.
- Do not upload dist files generated from an old version.
- PyPI versions are immutable and cannot be overwritten.
- TestPyPI does not allow filename reuse even for deleted files. If you get HTTP 400 filename reuse, bump version and rebuild.

Recovery commands for HTTP 400 filename reuse:
- Update pyproject.toml version (example: 0.1.0 -> 0.1.1)
- rm -rf dist build src/*.egg-info
- .venv/bin/python -m build
- .venv/bin/python -m twine check dist/*
- .venv/bin/python -m twine upload -r testpypi dist/jira_push_hook-0.1.1-py3-none-any.whl dist/jira_push_hook-0.1.1.tar.gz
- .venv/bin/python -m twine upload dist/jira_push_hook-0.1.1-py3-none-any.whl dist/jira_push_hook-0.1.1.tar.gz
